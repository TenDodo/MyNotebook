Linux Distros

A Linux distribution is an operating system made from a software collection that is based upon the Linux kernel and, often, a package management system.

Why are there so many distros?

Unlike Windows and Mac OS X Linux isn't corporate product. Different organizations and people are making many Linux Distributions. Different distros are using different software, package managers, system settings.

Sample Linux Distributions

Ubuntu

Difficulty

★

Ubuntu is one of the most popular distros. It's based on Debian and obviously uses the same package manager (APT). Ubuntu is very user friendly, has easy installation and very strong community support. Defaultly it uses GNOME desktop environment but you can choose alternative version with other DE: Kubuntu with KDE Plasma, Xubuntu with Xfce, Lubuntu with LXDE/LXQT.

Recomended system requirements (for Ubuntu 20.04.3 LTS):

  • 2 GHz dual core processor or better
  • 4 GB system memory
  • 25 GB of free hard drive space

Linux Mint

Difficulty

★

Linux Mint is popular Linux distro based on Ubuntu but it features better support for old hardware. You can download it with Cinnamon, MATE or Xfce desktop environment. It's very user friendly and stable and has similar setup to that of Windows.

Recomended system requirements (for Linux Mint 20.2, either Cinnamon, MATE, or XFCE edition):

  • 4 GB system memory
  • 20 GB recommended hard-drive space
  • Screen of minimum 1024×768 resolution

Manjaro

Difficulty

★

Manjaro is an open source operating system based on Arch Linux. It uses Pacman package manager and is able to use AUR (Arch User Repository) - community-driven repository for Arch users with tons of software. It also has great hardware support, easy kernels switching and huge community.

Recomended system requirements:

  • 2 GHz dual core processor or better
  • 4 GB system memory
  • 30 GB recommended hard-drive space

Pop!_OS

Difficulty

★

Pop!_OS is an open source operating system based upon Ubuntu, and featuring a GTK-based desktop environment known as COSMIC, which is based on GNOME. It provides full out-of-the-box support for both AMD and Nvidia GPUs. Unlike many other Linux distributions, it is not community-driven, although outside programmers can contribute, view and modify the source code.

Recomended system requirements:

  • 64-bit x86 processor
  • 4 GB system memory
  • 20 GB recommended hard-drive space

Debian

Difficulty

★★

Debian operating system for a wide range of devices. It's one of the oldest distributions and it's base for most other linux distros. It contain less software than Ubuntu by default and is less beginner friendly. Debian's strong point is that it is very lightweight.

Debian does not impose hardware requirements beyond the requirements of the Linux kernel and the GNU tool-sets. Therefore, any architecture or platform to which the Linux kernel, libc, gcc, etc. have been ported, and for which a Debian port exists, can run Debian.


Fedora

Difficulty

★★

Fedora is a Linux distribution developed by the Fedora Project which is sponsored primarily by Red Hat (an IBM subsidiary) with additional support and sponsors from other companies and organizations. Additional software is available from the software repositories and can be installed using the DNF package manager or GNOME Software. Majorly Fedora focuses on the new technology integration and innovation.

Recomended system requirements:

  • 2 GHz quad core processor
  • 4 GB system memory
  • 20 GB recommended hard-drive space

Arch Linux

Difficulty

★★★

Arch Linux attempts to have minimal distribution-specific changes, and therefore minimal breakage with updates, and be pragmatic over ideological design choices and focus on customizability rather than user-friendliness. Arch Linux has comprehensive documentation, which consists of a community wiki known as the ArchWiki.

Recomended system requirements:

  • x86-64 processor
  • 1 GB system memory
  • 20 GB recommended hard-drive space

Gentoo Linux

Difficulty

★★★

Gentoo Linux is a Linux distribution built using the Portage package management system. Unlike a binary software distribution, the source code is compiled locally according to the user's preferences and is often optimized for the specific type of computer. Precompiled binaries are available for some larger packages or those with no available source code.

Recomended system requirements:

  • processor i686 or later
  • 512 MB system memory
  • 2.5 GB recommended hard-drive space

Linux From Scratch

Difficulty

Harder than hard

Linux From Scratch (LFS) is not a specific distribution. This is a tutorial on how to build your own custom Linux system, entirely from source code with step-by-step instructions. The book is available freely from the Linux From Scratch site. The advantage of this system is its compactness. According to project official website it's not hard to get an LFS system installed under 100 MB. It's even possible to run LFS and Apache web server with total disk usage about 8 MB.

Requirements: Clean partition, working Linux system with required packages (Bash, Binutils, Bison, Bzip2, Coreutils, Diffutils, Findutils, Gawk, GCC, Glibc, Grep, Gzip, Linux Kernel)
