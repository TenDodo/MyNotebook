Linux filesystem

The file system in Linux is different from that in Windows. Unix-like systems use a standard that defines the placement of files and directories called Filesystem Hierarchy Standard (FHS). In the FHS, all files and directories appear under the root directory, even if they are stored on different physical or virtual devices.

Shorthand command referring to directories

/ - refers to the root directory.

~ - refers to the currently used user directory.

Directory structure

/ - Root directory of the entire file system hierarchy.

/bin - Contains essential binaries and executables that can be run from the command line at any time (e.g. cat, ls, cp).

/sbin - Essential system binaries that can be run by root user (e.g. fsck, init, route).

/lib - Libraries essential for the binaries in /bin and /sbin.

/lib32 or /lib64 - Alternate format essential libraries. These are typically used on systems that support more than one executable code format, such as systems supporting 32-bit and 64-bit versions of an instruction set.

/usr - Not essential read-only user data. Contains the majority of multi-user utilities and applications. It has its own /usr/bin, /usr/sbin, /usr/lib, /usr/lib<qual> directories. It also contain /usr/include (standard include files), /usr/local (tertiary hierarchy for local data, specific to this host, it also contains its own /usr/local/bin directory with manually compiled files), /usr/share (architecture-independent shared data), /usr/src (source code e.g. the kernel source code).

/etc - Host-specific system-wide configuration files. Many of files here are text-based config files that can be modified mostly with sudo privileges.

/home - User's home directories, containing saved files, personal settings, etc. Each user has its own directory here called by their username.

/boot - Files needed to boot the system (e.g. boot loader, kernels, initrd).

/dev - Device files. In this folder hardware and drivers are represented as regular files. Here we can find files like /dev/sda1 (physical disk partitions), /dev/tty (special file, representing the terminal for the current process), /dev/random (special file that serve as pseudorandom number generator).

/opt - Optional or add-on software.

/var - Variable files that will change as the operating system is being used (logs and cache files).

/tmp - Directory for temporary files that are often not preserved between system reboots.

/proc - This directory doesn't exist on the disk. It's being created every time the system starts up to keep track of running processes.
